- fix: 코드를 수정하거나 버그 수정

- chore : 그런트 작업 업데이트, 프로덕션 코드 변경 없음 (실제 프로덕션에 적용되지 않는 도구 변경이나 구성 변경)
- docs: 문서를 추가했을 때
- style: 코드 변경이 아닌 단순 포맷팅 변경 (새미콜론 제거 등)
- refactor: 코드를 리펙토링해서 개선했을 경우
- perf: 성능 개선
- test: 테스트코드를 추가한 경우
